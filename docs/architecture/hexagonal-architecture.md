# Hexagonal Architecture

## Principio fundamental

El dominio no depende de frameworks.

## Capas

### Domain
Lógica de negocio pura.

### Application
Casos de uso.

### Infrastructure
Persistencia, seguridad, integraciones.

### Interfaces
Controladores REST.

## Beneficios

- desacoplamiento
- testabilidad
- mantenibilidad
