# ADR-004: JWT for Authentication

## Estado
Aceptado

## Contexto
El sistema requiere un mecanismo de autenticación que:
- sea escalable,
- funcione con múltiples clientes,
- no dependa de sesiones en servidor.

## Decisión
Se adopta autenticación basada en **JWT (JSON Web Tokens)**.

El backend será responsable de:
- emitir tokens al autenticar,
- validar tokens en cada request,
- aplicar autorización basada en roles.

## Alternativas consideradas

### Sesiones en servidor
- Pros: simplicidad
- Contras: difícil escalabilidad, dependencia de estado

### OAuth2 completo
- Pros: estándar robusto
- Contras: complejidad innecesaria para este caso

## Consecuencias

### Positivas
- arquitectura stateless
- fácil integración con web y móvil
- escalabilidad horizontal

### Negativas
- gestión de expiración de tokens
- revocación más compleja

## Justificación
JWT permite desacoplar autenticación del estado del servidor, facilitando escalabilidad y soporte multi-cliente.
